Hash Algorithm

The blocks are hash using the SHA-256 Hash.

Hash Input:

The hash input contains the following fields all appended against each other. Each field is trimmed (leading and trailing white-space removed), but internal white space is kept.

  • Certificate Catgeory
  • Certificate Token
  • Certificate Url
  • Date Range
  • Degree Name
  • Description
  • Block Index
  • Instituition Name
  • Previous Hash
  • Timestamp
  • User Id

Example

  • Certificate Catgeory = Education
  • Certificate Token = ddd77eb1-27a1-493d-85de-bf494fb4267a
  • Certificate Url = https://www.credly.com/badges/ddd77eb1-27a1-493d-85de-bf494fb4267a?source=linked_in_profile
  • Date Range = October 2021
  • Degree Name = Google Data Analytics Professional Certificate
  • Description = Learned BigQueary, R, Tableau, Data Narratives/Presentation, Data Life Cycle, etc.
  • Block Index = 1
  • Instituition Name = Coursera
  • Previous Hash = 000001710634fb1c336cc7e35712c07975f3093afd88544f26ce9b4814186485
  • Timestamp = 1636993161
  • User Id = 3

Mining Algorithm

The mining alogrithm used is simple. In order for a block to be mined, each block's hash must start with five 0's.

By incrementing the block's nonce by 1, eventually a nonce is found where the block's hash starts with five 0's.
